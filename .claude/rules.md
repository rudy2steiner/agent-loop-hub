## 写作语言
所有生成的文档一律用中文撰写，包括 proposal.md、design.md、tasks.md、specs/**/*.md 等 openspec 产物。

## 文档一致性
修改任何 SDD 文档（proposal.md、design.md、tasks.md、specs/**/*.md）时，必须同步检查并更新所有相关文件，确保设计决策、模块名称、接口定义、技术选型在多个文档间保持一致，不允许出现各文件描述相互矛盾的情况。

## 任务并行化（tasks.md 编写规则）

生成 openspec tasks.md 时，必须按多 Agent 并行执行的方式组织任务结构：

1. **按依赖关系分层（Group）**：每个 Group 内的任务可以并行执行，Group 之间串行阻塞。
2. **标注并行单元（Agent）**：同一 Group 内，用 `### Agent X: 描述` 标注可并行的工作单元。
3. **文件隔离原则**：同一 Group 内的不同 Agent 必须触碰独立文件，避免合并冲突。
4. **合并粒度**：同一文件内的多个改动合并为一个任务，不要拆太细（如同一类的多个方法实现合为一条）。
5. **结构模板**：
   ```
   ## 1. 前置（串行）
   - [ ] 1.1 ...

   ## 2. 核心实现（并行: Agent A ‖ Agent B）
   ### Agent A: ...
   - [ ] 2.1 ...
   ### Agent B: ...
   - [ ] 2.2 ...

   ## 3. 验证（串行收尾）
   - [ ] 3.1 ...
   ```
6. **最大化并行窗口**：优先识别无依赖的独立文件改动，尽早并行分配，减少整体串行路径。
